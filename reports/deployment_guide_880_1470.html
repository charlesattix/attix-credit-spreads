Part 1: EXP-880 — Crisis Hedge V2 Ultra-Safe

What Is It?

EXP-880 is ML-filtered credit spreads on SPY with three layers of protection:

  1. ML Ensemble Signal Filter (from EXP-860) — 3-model ensemble (XGBoost + RandomForest + ExtraTrees) with P≥0.75 threshold. Only takes trades the ML models agree on.
  2. Regime Leverage (from EXP-840) — 2× base leverage that adapts to market regime (2.4× bull, 1.6× sideways, 0.6× bear, 0.2× crash).
  3. Crisis Hedge V2 — VIX-triggered delevering + put overlay + drawdown-controlled scaling. Never fully halts (min 20% scale).

Data Source

Backtested on COMPASS calibrated synthetic data, NOT raw IronVault data.
The COMPASS modules generate strategy return profiles calibrated to match the real IronVault options data patterns (same DTE distributions, IV surfaces, regime characteristics). However, individual trade entries are synthetic — not real historical option chains. This means:
  • Aggregate metrics (CAGR, Sharpe, DD) should be representative of real performance
  • Individual trade-level accuracy is approximate
  • Paper trading is ESSENTIAL to validate before live deployment
The existing EXP-400/401/503/600/700 strategies on main were backtested on real IronVault data and are currently paper trading. EXP-880 needs the same paper validation period.

Realistic Assessment of EXP-1470

Honest take: EXP-1470's numbers look incredible but deploying it requires significant infrastructure that doesn't exist yet:
  • The intraday mean reversion leg (20.5% of the portfolio) needs real-time 0-DTE options execution — the current scanner runs once per day at 10:00 ET. This is a fundamentally different execution paradigm.
  • The portfolio requires a multi-strategy orchestrator — something the current main.py scheduler wasn't designed for.
  • The 17.21 Sharpe ratio is based on combining strategies with near-zero correlation (0.033). In reality, correlations spike during stress — the validation tested ρ=0.5 and got 7.8% DD, which is fine but much worse than the base case.
  • Recommendation: Deploy EXP-880 first. It's a single strategy, uses existing infrastructure, and has a config ready. EXP-1470 is a medium-term goal that requires infrastructure buildout.
